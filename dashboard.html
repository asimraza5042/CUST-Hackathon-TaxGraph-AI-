🧠 Technical Architecture Stack
① Entity Resolution Pipeline
String distance: Levenshtein, Jaro-Winkler, Jaccard similarity for name matching. Blocking on CNIC prefix (first 5 digits) for O(n) scalability. Recall@5: 94%. LLM-assisted address normalization using llama-3.1-8B for Urdu/Roman transliteration. Handles "Muhammad Ahmed", "M. Ahmed", "Chaudhary M. Ahmed" → single resolved entity.
② Knowledge Graph Engine
Neo4j (production) or NetworkX (development). Nodes: Person, Vehicle, Property, UtilityMeter, FlightRecord, BankAccount, BusinessEntity. Edges: owns, registered_at, filed_in, declared_at, traveled, transacted_with. GNN layer: GraphSAGE + Graph Attention Network (2-layer) for node-level anomaly scoring.
③ TCDS Scoring Formula
Tax Compliance Deviation Score (0–100) = weighted sum of: asset-income gap (40%), utility-income ratio (20%), GNN anomaly score (25%), cross-DB inconsistencies (15%). Peer-group normalized per city/income tier. SHAP values generated per feature for explainability.
④ XAI Audit Trail
Every flagged entity gets: (a) Step-by-step evidence chain, (b) SHAP contribution per data source, (c) Human-readable reason for each flag, (d) FBR Section 122 notice template. Output: structured JSON → PDF → FBR portal API. Legally defensible in audit proceedings.
